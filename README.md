# 职场开局

“职场开局”副标题为“AI新人90天成长副本”，是一个以成长证据为基础的AI Native新人适应与独立交付能力验证系统。它将智能画像诊断、90天路径、互动闯关、成果证据、规则评价、导师验收和HR批次洞察连接成完整链路。

这是可本地运行并公开部署的Streamlit概念演示产品，采用浅色多巴胺成长游戏界面与轻量企业级HR看板，不代表腾讯或其他企业的内部正式系统。

## 产品定位

AI负责路径生成、过程监测、成长证据评价、风险识别、建议整理和内容草案；新人、导师与HR负责事实核验、业务判断、责任承担和最终决策。默认模式为“本地规则稳定模式”，没有API密钥时使用内置规则与Demo模拟数据稳定运行；配置API后自动进入“AI增强模式”。

## 三类用户

- 新人：获得个性化画像、AI能力诊断、成长策略、90天任务和动态复盘。
- 导师：查看能力变化、延期任务、主要障碍、带教建议、一对一问题和待验收任务。
- HR：查看批次风险、岗位进度、AI成熟度、导师负载、高频障碍和组织培养建议。

## 核心功能

- 双入口首页：一键体验完整90天，或创建自己的成长副本。
- 五步智能画像：覆盖岗位、业务场景、AI能力、学习方式和90天目标。
- AI诊断报告：展示能力雷达、AI协作成熟度、优势、差距、风险和带教强度。
- 成长世界地图：包含新手营地、业务探索区、协作试炼场、真实项目区和成果答辩厅。
- 成长证据链：每项任务可提交成果摘要、人机分工、AI验证过程、问题和自评完成度。
- 四维成果评价：本地规则引擎评价业务相关性、证据质量、AI验证能力和独立交付能力。
- 导师成果验收：支持通过验收、退回补充和调整任务范围，证据结果优先于Demo手动状态。
- 30—60—90阶段闸门：分别验证完成适应、形成贡献和独立交付。
- 三档任务强度：轻量生成7项核心任务，标准生成10项任务，挑战生成10项核心任务和2项挑战任务。
- 人机责任边界：每项任务明确人负责、AI负责、必须人工验证和禁止输入AI的信息。
- 每周动态复盘：同步任务状态、形成新人行动卡，并在自动调整开启时写入动态任务。
- 导师面板：将过程信号转成可执行的带教动作和验收清单。
- HR驾驶舱：使用20名Demo模拟新人展示批次培养洞察，小鹅的阶段、风险、证据和导师动作与自动演示同步。
- 三方行动卡：复盘或证据评价后同步形成新人、导师和HR行动建议。
- 轻量导出：支持导出个人成长报告、最近周复盘、导师沟通卡和HR培养简报。
- 全站AI互动：首页今日成长建议、画像解读、诊断摘要、任务配置器、复盘教练、自动演示旁白、导师追问生成器和HR组织洞察。

## 互动闯关

侧边栏进入“互动体验 / 互动闯关”后，会先看到《职场开局》的玩法介绍页。当前已完成第一幕和第二幕，形成从开局判断到方案救火的连续剧情闭环。

- 第一幕“开局判断”：通过业务理解和AI数据核验观察玩家基础行为倾向。
- 第二幕“方案救火”：通过AI方案审查、提示词改写挑战、任务优先级和证据包组装，将一份存在问题的AI初稿修复为可评审方案。
- 成长搭子“芽芽”：第二幕中会根据第一幕风格获得初始特质、食物或提示券。完成挑战后可获得核验莓、行动饼、证据星或复盘饼，喂食后推动芽芽成长并解锁有限提示或轻量恢复。
- 低分补救：任一小游戏低于50分时不会形成死路，玩家可通过复盘饼触发30秒复盘问题；答对可降低少量风险，但不修改原始分数。
- 第三幕“协作试炼”：通过需求拉齐、范围取舍和突发事件盲盒，形成团队可执行版本。
- 第四幕“成果验收”：通过证据装配、导师连续追问、主观答辩和成长基地，完成终局评审闭环。

互动闯关结果、芽芽成长值、食物、表情和提示券只用于游戏进度与轻量反馈，不用于正式人才评价。数据仅保存在当前会话的`st.session_state.game`中，不会修改成长证据链、自动演示、HR批次模拟数据或正式成长计划。

## 自动演示

首页点击“一键体验完整90天”进入自动演示。

- “开始”：使用片段级短间隔刷新，从当前进度自动连续播放。
- “暂停”与“继续”：控制自动播放。
- “播放下一幕”：适合现场讲解时逐幕推进。
- “加速”：在1×、2×和4×之间切换。
- “重置”：恢复到第0周。

自动演示包含画像完成、路径生成、任务延期、中风险识别、计划调整、导师介入、跨团队协作、首次证据提交、补证要求、导师退回、新人补证、导师通过、评审和成果答辩等15个事件。

现场展示推荐使用“播放下一幕”，便于讲解每个关键事件并保持节奏稳定；自动播放适合快速预览完整链路。自动播放会在最后一个事件停止，不会无限循环。旧版Streamlit不支持片段刷新时，应用会自动降级为逐幕播放。

## 3分钟演示主线

1. 首页点击“一键体验完整90天”，查看小鹅画像、诊断和90天成长地图。
2. 使用“播放下一幕”推进到第3周，展示延期、中风险、计划调整和导师介入。
3. 继续推进到第8周，展示活动方案首次证据、四维评价和“待补充”状态。
4. 展示导师追问、新人补充玩家反馈与数据口径、导师确认通过和能力指标变化。
5. 打开HR驾驶舱，展开小鹅详情，查看风险从中降为低以及组织行动建议。
6. 返回成长地图，导出个人90天成长报告。

## 本地运行

建议使用Python3.10或更高版本。

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
streamlit run app.py
```

浏览器通常访问`http://localhost:8501`。

## 公网部署

可使用Streamlit Community Cloud公开部署：

1. 在GitHub创建仓库并上传以下文件：
   - `app.py`
   - `ai_runtime.py`
   - `game_content.py`
   - `game_engine.py`
   - `game_ui.py`
   - `requirements.txt`
   - `README.md`
2. 登录[Streamlit Community Cloud](https://share.streamlit.io/)并授权访问该仓库。
3. 点击“Create app”，选择仓库和分支。
4. 将入口文件设置为`app.py`并点击“Deploy”。

如果缺少`ai_runtime.py`，AI增强和本地fallback入口会导入失败；如果缺少`game_ui.py`、`game_engine.py`或`game_content.py`，互动闯关无法运行。

当前版本不需要数据库、登录系统、大模型API密钥或Secrets。若不配置API，仍可完整演示所有页面与互动闯关。配置`AI_NATIVE_LLM_API_KEY`、`AI_NATIVE_LLM_ENDPOINT`、`AI_NATIVE_LLM_MODEL`后启用AI增强模式。

## 可选AI增强

默认运行模式为“本地规则稳定模式”。如需连接OpenAI兼容的对话接口，可在本地环境变量或Streamlit Secrets中配置：

```toml
AI_NATIVE_LLM_API_KEY="your-key"
AI_NATIVE_LLM_ENDPOINT="https://your-provider.example/v1/chat/completions"
AI_NATIVE_LLM_MODEL="your-model"
```

配置完整时页面显示“AI增强模式”。大模型仅优化补证建议、导师追问和HR培养简报；分数、任务状态、阶段闸门、安全红线和导师触发仍由本地规则引擎决定。接口超时、调用失败或响应解析失败时会自动返回本地规则结果。

当前AI增强可用于：

- 画像分析和主观成长问题解读。
- 诊断报告摘要和不同口吻改写。
- 本周成长路径配置建议。
- 每周复盘反馈、导师同步话术和HR风险提示。
- 自动演示旁白与随机成长剧本解释。
- 导师追问生成和HR组织洞察简报。
- 互动闯关中的提示词优化和导师主观答辩反馈。

无论是否配置API，AI输出都只用于辅助分析和内容生成，最终判断需由本人、导师或HR确认。

## Demo数据声明

- 自动演示中的“小鹅”画像、成长事件、任务结果和导师建议均为Demo模拟数据。
- HR驾驶舱中的20名新人、风险、岗位分布、导师负载和培养简报均为Demo模拟数据。
- 以上内容不代表腾讯真实员工、内部数据、制度或正式产品。
- 成果评分由本地规则引擎模拟生成，仅用于成长辅助，最终评价由导师和HR确认。

## AI使用安全

个人隐私、账号密钥、未公开经营数据、商业机密、受限代码和未经授权的内部材料不得输入AI。AI输出必须经过事实、数据口径、用户结论和业务责任人的人工验证。

## 项目文件

- `app.py`：核心应用、规则引擎、演示状态机和全部页面。
- `app_backup.py`：本次产品级重构前的稳定版本备份。
- `app_dark_backup.py`：浅色主题改造前的深色新版备份。
- `app_evidence_backup.py`：本轮最终稳定性修复前的成长证据链版本备份。
- `app_before_game.py`：互动闯关第一阶段开发前的稳定备份。
- `app_before_ui_navigation.py`：本轮UI与导航重构前的稳定备份。
- `game_content.py`：互动闯关场景、显示映射和静态定义。
- `game_engine.py`：不依赖Streamlit的互动闯关纯逻辑。
- `game_ui.py`：互动闯关Streamlit页面。
- `ai_runtime.py`：可选大模型运行层；无配置时自动返回本地规则兜底结果。
- `tests/test_game_engine.py`：互动闯关纯逻辑单元测试。
- `requirements.txt`：Streamlit与Plotly依赖。
- `README.md`：产品与部署说明。

## 当前限制

- 数据只保存在当前浏览器会话的`st.session_state`中，刷新或重启服务后不会长期保留。
- 个性化计划与诊断由本地规则生成，不等同于真实组织人才评估。
- 成果链接只作为文本保存，不上传或存储真实文件。
- HR驾驶舱尚未连接企业数据源，导师面板也不支持跨用户协作。
- 自动演示使用Streamlit片段级刷新；网络延迟较高时，逐幕播放比连续播放更适合正式讲解。
- 可选AI增强采用通用OpenAI兼容接口，具体模型能力、费用、数据处理和服务可用性由所配置的供应商决定。
